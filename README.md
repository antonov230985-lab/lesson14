# Урок 14 — Cloud-driven Data Cleaning Pipeline

Проект развивает логику урока 13 в сторону модульного production-пайплайна: входной Excel берется из облака, преобразуется в локальные CSV, обрабатывается по задачам и возвращается в виде одного итогового `xlsx`.

## Что уже реализовано

- Разделение на изолированные модули-«черные ящики»:
  - `cloud/*` - клиенты Яндекс.Диска и Google Drive.
  - `cloud_io/*` - download/prepare и upload модули.
  - `processing/*` - нормализация и 5 задач очистки.
  - `sync/*` - state store, детектор изменений, цикл обработки.
- Обработка входного файла `homework_lesson13.xlsx` через промежуточный CSV-бандл (по листам).
- Сборка итогового файла `lesson14_all_tasks_clean.xlsx`.
- Непрерывный оркестратор в `app.py` для режима мониторинга изменений.

## Что будет делать в перспективе

- Полный режим near-real-time синхронизации:
  - периодическая проверка изменений исходного файла в обоих облаках;
  - выбор актуального источника;
  - автоматическая выгрузка результата обратно на Яндекс.Диск и Google Drive.
- Более надежный fallback (недоступность одного облака, временные ошибки API, ретраи с backoff).
- Расширяемая схема обработки: новые задачи можно добавлять отдельными модулями в `processing/tasks`.
- Базовый аудит обработки (логи, версии исходного файла, диагностика ошибок).

## Локальный запуск без облака (тестовый режим)

Этот режим делает ровно то, что ты просил: берет `homework_lesson13.xlsx`, сохраняет листы в CSV, прогоняет основным пайплайном и создает очищенный итоговый файл в текущей папке.

```bash
python local_run.py
```

Результат:

- `lesson14_all_tasks_clean.xlsx`

Временные файлы создаются во временной папке и автоматически удаляются после выполнения.

## Запуск cloud-режима

```bash
python app.py
```

Перед запуском нужно задать переменные окружения из `config.py`:

- `YANDEX_TOKEN`
- `GOOGLE_SERVICE_ACCOUNT_FILE`
- `GOOGLE_DRIVE_FILE_ID`
- `YANDEX_INPUT_PATH`
- `YANDEX_OUTPUT_PATH`
- `GOOGLE_OUTPUT_NAME`
- `POLLING_SECONDS`
- `STATE_FILE`
- `TEMP_ROOT`

## Зависимости

- Python 3.10+
- `pandas`
- `openpyxl`
- `requests`
- `google-api-python-client`
- `google-auth`

Установка:

```bash
pip install -r requirements.txt
```
